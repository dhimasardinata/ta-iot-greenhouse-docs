# gateway/.gitignore

File ini mengatur file apa saja di folder firmware gateway yang tidak perlu masuk ke penyimpanan kode.

## Metadata File

| Item | Nilai |
|---|---|
| Source file | `gateway/.gitignore` |
| Komponen | Config |
| Level | Pemula |
| Status | Drafted |
| Terakhir diperiksa | 2026-05-19 |

## Kenapa File Ini Ada

Saat firmware gateway dibangun, PlatformIO dan editor bisa membuat file sementara. File sementara seperti cache build, database index editor, dan arsip lokal tidak perlu dibaca sebagai source utama sistem TA. File `.gitignore` mencegah file seperti itu ikut tercampur dengan firmware yang benar-benar dirawat.

## Isi yang Terlihat

File ini mengabaikan:

- `.pio`
- `.vscode/.browse_c_cpp.db*`
- `.vscode/c_cpp_properties.json`
- `.vscode/launch.json`
- `.vscode/ipch`
- `Arsip.zip`

## Data Masuk dan Keluar

File ini tidak menerima data runtime dan tidak menghasilkan data runtime. Efeknya hanya muncul saat Git memutuskan file mana yang perlu dilacak.

## Kapan Dipakai

File ini dipakai saat developer menjalankan perintah Git, misalnya melihat status perubahan atau menyiapkan commit. Firmware gateway tetap bisa berjalan tanpa membaca file ini di perangkat.

## Error yang Mungkin Terjadi

- Jika `.pio` tidak diabaikan, hasil build bisa ikut masuk ke perubahan repository dan membuat dokumentasi/source terlihat kotor.
- Jika arsip lokal seperti `Arsip.zip` tidak diabaikan, file besar atau duplikat bisa ikut terbawa.
- Jika konfigurasi editor yang penting ikut diabaikan, developer lain mungkin perlu membuat ulang konfigurasi sendiri.

## Bagian untuk Pemula

Anggap file ini sebagai daftar "jangan masukkan ke catatan resmi project". Yang disimpan adalah source dan konfigurasi penting; yang diabaikan adalah hasil kerja sementara.

## Bagian Advanced

File ini hanya berlaku di area gateway. Karena root repository pada snapshot ini bukan repository Git tunggal, efek akhirnya bergantung pada repository mana yang benar-benar dipakai saat publish.

## Hubungan ke Sistem TA

File ini menjaga folder gateway tetap fokus pada firmware dan konfigurasi yang relevan, bukan output build atau cache editor.

Lanjutkan ke [gateway/partitions_custom.csv](./partitions_custom.csv.md).
