## Kenapa File Ini Ada

Pre-commit membantu mencegah perubahan yang belum rapi masuk ke Git. Pemeriksaan berjalan sebelum commit dibuat, sehingga kesalahan sederhana bisa ditangkap lebih cepat.

## Hook yang Dipakai

| Repo hook | Hook | Fungsi |
|---|---|---|
| `pre-commit/pre-commit-hooks` | `trailing-whitespace` | Menghapus spasi sisa di akhir baris. |
| `pre-commit/pre-commit-hooks` | `end-of-file-fixer` | Memastikan file berakhir rapi. |
| `pre-commit/pre-commit-hooks` | `check-yaml` | Memeriksa file YAML. |
| `pre-commit/pre-commit-hooks` | `check-added-large-files` | Mencegah file besar tidak sengaja masuk. |
| `pocc/pre-commit-hooks` | `clang-format` | Merapikan C/C++. |
| `pocc/pre-commit-hooks` | `cppcheck` | Menganalisis C/C++. |
| `local` | `pio-check` | Menjalankan `pio check --fail-on-defect=medium`. |

## Hal yang Perlu Diperhatikan

`clang-format` di hook memakai argumen `--style=Google`, sedangkan `node/.clang-format` berisi beberapa aturan tambahan dari gaya Google. Jika hasil format terasa tidak sama, bagian ini perlu dicek.

## Error yang Mungkin Terjadi

- Jika PlatformIO belum terpasang, hook `pio-check` gagal.
- Jika `cppcheck` menemukan banyak hal dari dependency, perlu pastikan exclude sudah benar.
- Jika hook lambat, developer bisa tergoda melewati pre-commit.

## Bagian untuk Pemula

File ini seperti pemeriksa sebelum menyimpan perubahan ke riwayat proyek. Kalau ada hal sederhana yang salah, commit ditahan dulu sampai diperbaiki.

## Bagian Advanced

`pio-check` hanya dijalankan untuk file C/C++ lewat pola `\\.(cpp|h|c|hpp)$` dan `pass_filenames: false`. Jadi command memeriksa proyek, bukan hanya file yang berubah.

## Hubungan ke Sistem TA

Pre-commit menjaga firmware node tetap rapi sebelum perubahan masuk ke repository, sehingga proses pengerjaan TA lebih mudah diaudit.
