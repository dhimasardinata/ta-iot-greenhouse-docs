# node/.lizardrc

File ini mengatur Lizard untuk mengukur kompleksitas kode firmware node.

## Metadata File

| Item | Nilai |
|---|---|
| Source file | `node/.lizardrc` |
| Komponen | Config |
| Level | Menengah |
| Status | Drafted |
| Terakhir diperiksa | 2026-05-19 |

## Kenapa File Ini Ada

Lizard membantu melihat fungsi yang terlalu panjang atau terlalu bercabang. Pada firmware, fungsi yang terlalu rumit lebih sulit diuji dan lebih rawan bug.

## Batas yang Dipakai

| Aturan | Nilai |
|---|---:|
| `length_threshold` | 50 |
| `cyclomatic_complexity_threshold` | 12 |
| `max_nloc` | 50 |

## Folder yang Dikecualikan

Lizard tidak memeriksa:

- `.pio/*`
- `.git/*`
- `build/*`
- `node_modules/*`

Ini mencegah hasil ukur tercampur dengan dependency dan output build.

## Error yang Mungkin Terjadi

- Jika ambang terlalu ketat, refactor kecil bisa menghasilkan banyak peringatan.
- Jika ambang terlalu longgar, fungsi besar bisa tidak segera terlihat.
- Jika folder source baru tidak tercakup, hasil Lizard kurang lengkap.

## Bagian untuk Pemula

File ini seperti batas ukuran paragraf untuk kode. Jika satu fungsi terlalu panjang atau terlalu banyak cabang, file ini membantu menandainya.

## Bagian Advanced

Kompleksitas siklomatik `12` relatif ketat untuk firmware state machine. Temuan Lizard harus dibaca bersama konteks, terutama untuk parser command dan alur OTA.

## Hubungan ke Sistem TA

Node menjalankan sensor, cache, upload, dan terminal. Mengontrol kompleksitas membantu menjaga bagian-bagian itu tetap mudah dirawat.
