# node/package-lock.json

File ini mengunci versi dependency NPM untuk tooling node.

## Metadata File

| Item | Nilai |
|---|---|
| Source file | `node/package-lock.json` |
| Komponen | Config |
| Level | Menengah |
| Status | Drafted |
| Terakhir diperiksa | 2026-05-19 |

## Kenapa File Ini Ada

`package.json` menyebut dependency, sedangkan `package-lock.json` menyimpan versi persis yang dipakai. Ini membuat hasil `npm ci` lebih konsisten di komputer lokal dan CI.

## Dependency yang Dikunci

| Package | Versi | Catatan |
|---|---:|---|
| `chokidar` | `5.0.0` | Dipakai untuk watcher file. |
| `crypto-js` | `4.2.0` | Dipakai untuk kebutuhan kriptografi JavaScript. |
| `readdirp` | `5.0.0` | Dependency dari `chokidar`. |

`chokidar` dan `readdirp` meminta Node `>= 20.19.0`.

## Error yang Mungkin Terjadi

- Jika Node terlalu lama, dependency bisa gagal berjalan.
- Jika `package.json` berubah tetapi lockfile tidak diperbarui, `npm ci` bisa gagal.
- Jika integritas package berubah, NPM akan menolak install.

## Bagian untuk Pemula

File ini seperti nota belanja yang sangat rinci. Ia memastikan komputer lain memasang versi alat bantu yang sama.

## Bagian Advanced

Lockfile ini kecil karena dependency NPM proyek juga kecil. Jalur build firmware utama tetap berada di PlatformIO, bukan NPM.

## Hubungan ke Sistem TA

Dependency NPM mendukung watcher dan asset/tooling web node. Lockfile menjaga tooling itu tidak berubah diam-diam.
