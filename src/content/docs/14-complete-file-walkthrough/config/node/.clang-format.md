# node/.clang-format

File ini mengatur gaya penulisan kode C/C++ pada firmware node.

## Metadata File

| Item | Nilai |
|---|---|
| Source file | `node/.clang-format` |
| Komponen | Config |
| Level | Menengah |
| Status | Drafted |
| Terakhir diperiksa | 2026-05-19 |

## Kenapa File Ini Ada

Firmware node punya banyak file C++ di `include/`, `src/`, dan `lib/NodeCore/`. File ini membantu semua file itu rapi dengan aturan yang sama, sehingga perbedaan gaya antar kontributor tidak membuat review sulit.

## Aturan Utama

| Aturan | Nilai di file | Arti praktis |
|---|---|---|
| Bahasa | `Cpp` | Format dipakai untuk kode C++. |
| Standar | `c++20` | Formatter mengerti pola C++20. |
| Dasar gaya | `Google` | Mulai dari gaya Google lalu disesuaikan. |
| Indentasi | `2` spasi | Blok kode masuk ke kanan dua spasi. |
| Tab | `Never` | File tidak memakai tab untuk indentasi. |
| Batas kolom | `120` | Baris panjang dipotong setelah kira-kira 120 karakter. |

## Include dan Namespace

File ini mengaktifkan `SortIncludes: CaseSensitive` dan `IncludeBlocks: Regroup`. Artinya daftar `#include` akan diurutkan dan dikelompokkan ulang.

`NamespaceIndentation: All` dan `FixNamespaceComments: true` membuat isi namespace tetap menjorok dan penutup namespace diberi komentar otomatis.

## Pointer, Reference, dan Blok

Pointer memakai gaya `Type* nama`, bukan `Type *nama`. Kurung kurawal memakai gaya menempel di baris yang sama. `if` dan loop pendek tidak dipadatkan dalam satu baris, sehingga lebih mudah dibaca saat debugging.

## Error yang Mungkin Terjadi

- Jika versi `clang-format` terlalu lama, beberapa opsi seperti `IndentRequiresClause` bisa tidak dikenal.
- Jika format otomatis belum dijalankan, hasil diff bisa penuh perubahan gaya.
- Jika file vendor ikut diformat, perubahan bisa melebar ke kode yang bukan milik proyek.

## Bagian untuk Pemula

File ini seperti aturan merapikan tulisan. Isinya tidak mengubah cara alat bekerja di greenhouse, tetapi membuat kode lebih mudah dibaca dan lebih konsisten.

## Bagian Advanced

Aturan `BinPackArguments: false` dan `BinPackParameters: false` membuat argumen panjang dipisah per baris. Ini biasanya menghasilkan diff yang lebih bersih saat satu parameter ditambah atau dihapus.

## Hubungan ke Sistem TA

File ini menjaga kualitas kerja tim saat mengubah firmware node, terutama karena node berisi banyak modul kecil untuk sensor, penyimpanan, OTA, terminal, dan komunikasi cloud.
