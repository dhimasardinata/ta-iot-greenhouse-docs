# node/.hintrc

File ini mengatur webhint untuk pemeriksaan halaman web pada proyek node.

## Metadata File

| Item | Nilai |

## Kenapa File Ini Ada

Node punya halaman lokal seperti portal, terminal, status sukses, dan update. `webhint` dapat membantu memeriksa kualitas HTML/CSS/JS halaman tersebut.

## Isi Utama

File ini memakai konfigurasi `development`, lalu mematikan beberapa hint:

- `no-inline-styles`
- `axe/language`
- `meta-charset-utf-8`

## Makna Praktis

Mematikan `no-inline-styles` berarti halaman yang memakai style langsung di HTML tidak langsung dianggap masalah. Ini masuk akal untuk firmware kecil karena halaman sering dibuat ringkas.

## Error yang Mungkin Terjadi

- Jika hint terlalu banyak dimatikan, masalah aksesibilitas atau HTML bisa luput.
- Jika hint terlalu ketat, halaman firmware kecil bisa banyak gagal karena kompromi ukuran.

## Bagian untuk Pemula

File ini seperti aturan pemeriksaan halaman web. Beberapa peringatan dimatikan agar cocok dengan kebutuhan halaman kecil di perangkat.

## Bagian Advanced

Konfigurasi ini condong ke mode pengembangan, bukan audit aksesibilitas penuh. Untuk publikasi dashboard yang lebih luas, pemeriksaan aksesibilitas bisa dibuat lebih ketat.

## Hubungan ke Sistem TA

Halaman lokal node dipakai untuk portal WiFi, terminal, dan update. Pemeriksaan web membantu menjaga halaman tetap bisa dibuka dan tidak mudah rusak.
