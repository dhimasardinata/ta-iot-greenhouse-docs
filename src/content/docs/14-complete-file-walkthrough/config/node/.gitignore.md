# node/.gitignore

File ini menentukan file node mana yang tidak perlu masuk Git.

## Metadata File

| Item | Nilai |
|---|---|
| Source file | `node/.gitignore` |
| Komponen | Config |
| Level | Dasar |
| Status | Drafted |
| Terakhir diperiksa | 2026-05-19 |

## Kenapa File Ini Ada

Tidak semua file di folder node adalah source yang perlu disimpan. Ada hasil build, cache editor, dependency hasil install, sertifikat lokal, dan file bantuan yang berubah-ubah di komputer masing-masing. File ini mencegah file seperti itu ikut tersimpan sebagai source proyek.

## Pola yang Diabaikan

| Pola | Arti |
|---|---|
| `.pio` | Output dan cache PlatformIO. |
| `.vscode/.browse.c_cpp.db*` | Database browsing C/C++ dari editor. |
| `.vscode/c_cpp_properties.json` | Konfigurasi IntelliSense yang dibuat PlatformIO. |
| `.vscode/launch.json` | Konfigurasi debug lokal. |
| `.vscode/ipch` | Cache C/C++ editor. |
| `node_modules/` | Dependency NPM hasil install. |
| `var/` dan `coverage/` | Output runtime/test lokal. |
| `tools/prompt/` dan `tools/context/` | Hasil kumpulan konteks/prompt. |
| `compile_commands.json` | Database compile hasil generator. |
| `*.key` dan `*.pem` | Sertifikat atau kunci lokal. |

## Hal yang Perlu Diperhatikan

File ini memakai pola `.github\instructions\codacy.instructions.md` dengan backslash. Pada sistem Linux, path repository biasanya memakai slash `/`, sehingga pola ini perlu diperiksa jika tujuannya benar-benar mengabaikan `.github/instructions/codacy.instructions.md`.

## Error yang Mungkin Terjadi

- Jika file hasil build ikut Git, repository bisa membesar dan sulit direview.
- Jika file penting terlalu luas di-ignore, source bisa tidak sengaja hilang dari commit.
- Jika sertifikat lokal tidak di-ignore, data sensitif bisa ikut tersebar.

## Bagian untuk Pemula

File ini seperti daftar barang yang tidak perlu dimasukkan ke arsip proyek. Source utama disimpan, sedangkan sampah build dan file rahasia lokal ditinggal.

## Bagian Advanced

`compile_commands.json` di-ignore walau masuk coverage sebagai config penting. Artinya file tersebut dapat dibuat ulang dari build lokal dan tidak wajib menjadi source permanen.

## Hubungan ke Sistem TA

Dengan `.gitignore`, repository node tetap berisi source firmware dan konfigurasi penting, bukan cache atau kunci lokal yang dapat mengganggu audit TA.
